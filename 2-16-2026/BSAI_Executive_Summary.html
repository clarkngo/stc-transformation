BSAI Program Revision - Executive Summary

Monday Leadership Meeting - February 2026

THE PROBLEM

Current BSAI program has 3 critical gaps that prevent graduates from meeting AI Engineer Roadmap 2026 standards:

1. Math Stops at Precalculus ❌

  • Cannot understand gradient descent (Deep Learning)
  • Cannot manipulate matrices (Neural Networks)
  • Cannot quantify uncertainty (Model Evaluation)

2. Missing Machine Learning Foundation ❌

  • Jump from "Fundamentals" directly to "Generative AI"
  • Skip classical ML (Random Forests, SVMs, Ensemble methods)
  • Students don't understand what LLMs are built on

3. R-Based Statistics (Industry uses Python) ❌

  • 95% of AI production code is Python
  • Students learn NumPy in Year 3, should learn Year 1
  • Wasted time re-learning tools

THE SOLUTION

CHANGE 1: Add Math for AI (15 credits)

NEW COURSES:

MATH 151 - Calculus I (Gradients, Optimization)           5 credits
MATH 251 - Linear Algebra (Matrices, Transformations)    5 credits  
MATH 261 - Probability & Statistics for AI               5 credits

WHERE: Replace 15 credits of General Education electives

DELIVERY: Computational approach using Python (not pencil-paper)

CHANGE 2: Python-First Programming Pathway

CURRENT:

CS 132 (Java) → CS 251 (R Statistics) → AI courses

REVISED:

CS 132 (Python I) → CS 133 (Python II) → CS 251 (Python: NumPy/SciPy) → AI courses

CS 251 BECOMES: "Computational Mathematics & Statistics"

  • Week 1-5: Linear Algebra with NumPy
  • Week 6-10: Statistics with SciPy/Pandas

CHANGE 3: Core Curriculum Restructure

REMOVE FROM CORE:

❌ AI 370 Generative AI (move to Depth of Study electives)

ADD TO CORE:

✅ DS 479 Machine Learning and Data Mining (5 credits)

RATIONALE:

Students MUST master classical ML before advanced topics:

Fundamentals → Machine Learning → Deep Learning → Generative AI
(AI 320)      (DS 479)           (DS 489)        (AI 370)

ANSWERS TO KEY MEETING QUESTIONS

Q1: How will this impact CityU MS DS program?

ANSWER: POSITIVE IMPACT

✅ BSAI graduates arrive with:

  • Machine Learning foundation (DS 479)
  • Python fluency (NumPy, Pandas, PyTorch)
  • Cloud computing experience (IT 434)

✅ MS DS can:

  • Waive/accelerate foundational courses
  • Focus on advanced specializations (Causal Inference, Bayesian Methods)
  • Reduce remedial instruction time

NO NEGATIVE IMPACT—actually strengthens the pipeline

Q2: Do CY (Cybersecurity) people need DS Algo, ML, DL?

ANSWER: YES (2026 Threat Landscape)

Modern Cyber Threats Require AI Understanding:

  • AI-generated phishing emails (LLM detection)
  • Deepfake detection (Generative AI forensics)
  • Malware behavior analysis (ML classifiers)
  • Anomaly detection in OT/ICS systems (Pattern recognition)

BSAI Already Addresses This:

  • IS 345 Cybersecurity (core)
  • IS 471 Cyber and AI Ethics (addresses adversarial ML, deepfakes)

Optional: Students can specialize in "AI Security" within Depth of Study

Q3: Can CY survive with these core requirements?

ANSWER: YES—DIFFERENT CORE REQUIREMENTS

Recommendation: Create BSCY (Cybersecurity) with modified core:

BSAI Core BSCY Alternative
DS 479 Machine Learning IS 445 Network Security
AI 320 Fundamentals of AI IS 450 Penetration Testing
IT 434 Cloud Computing IS 460 Cloud Security (different focus)

Key Principle:

  • BSAI = "Building AI systems"
  • BSCY = "Securing all systems (including AI)"
