Strategic picture (STC · ADK Masterclass)

Source of truth: The public video is Google’s ADK Masterclass track; Example #1 is Building Your First Agent — a greeting agent that captures the user’s name and responds to it, while introducing Agent, Session, and Runner as the three core building blocks.

Democratization thought leadership. In content-strategy terms, the school’s play is to claim authority by going wider, not by out-explaining the masterclass line-for-line to senior developers. You open the same ideas to everyone those developers report to, collaborate with, and pitch to — a materially larger audience — using workplace scenarios and plain-language “what could go wrong?” framing instead of competing on terminal speed.

Why Example #1 is the right anchor. The greeting agent is small in code but rich in the questions non-technical stakeholders actually ask: What does the agent “want”? Who wrote its personality? What if instructions are sloppy? What if the goal were wrong? You can run that thought experiment in a meeting-room story without showing code and often land more memorably than another screen recording.

Biggest execution risk: the anchor persona. Courses like CS50 are inseparable from a consistent on-camera teacher over years. For STC-branded follow-on content, the Dean (or one named faculty voice) needs to own that role with a multi-year commitment — production polish and channel tactics are secondary to showing up reliably with the same trusted face and voice.

Working title (technical cut): Example #1 (Basic Agent) — 10 concepts the crash course assumes you know

1. Learning outcomes

By the end, the viewer can:

  1. Name Google’s three lesson blocks — Agent, Session, Runner — and say what each owns in plain language.
  2. Explain what an ADK agent is versus a one-off “call Gemini from Python.”
  3. Navigate the Basic Agent folder and name the files that define model + behavior + entrypoint.
  4. Create/activate venv, install deps, and configure .env from .env.example in the correct folder.
  5. Run the example and diagnose at least three common failures (key, Python version, wrong directory).
  6. Describe sessions at a high level (why multi-turn works without manual history concatenation).
