AI 510: Artificial Intelligence in Cloud Computing - Proposed Assessments

Course Description: Explores deploying, managing, and scaling AI solutions using major cloud platforms (e.g., AWS, Azure, GCP). Covers serverless functions, containerization, and managed AI services.

1. Applied Case Studies / Conceptual Assessment (40%)

Example Scenario: Cloud AI Strategy for a Startup

Prompt: A nascent startup, "GreenThumb AI," aims to provide drone-based agricultural insights using computer vision models trained on satellite imagery. They have limited budget but anticipate rapid growth. As their lead cloud architect, prepare a cloud infrastructure proposal (5-page max) and a cost-analysis spreadsheet for a specific major cloud provider (e.g., AWS). Your proposal must:

  • Recommend specific cloud services for data ingestion (e.g., S3, Data Lake), model training (e.g., SageMaker, Vertex AI), and inference deployment (e.g., Lambda, AKS).
  • Justify your choices based on cost-efficiency, scalability, and ease of management for a small team.
  • Outline key security considerations and compliance requirements for agricultural data.

Why AI-Resistant? Requires applying broad cloud and AI knowledge to a specific startup's constraints (budget, growth, data type) on a named platform. While AI can list services, selecting and justifying them for a nuanced scenario, along with a detailed cost analysis, demands human expertise.

Example Viva Questions (Post-Submission)

  • "You chose AWS SageMaker for model training. What specific **Hands-On Skills (HOS)** would a data scientist need to containerize their custom training script for SageMaker?"
  • "Explain a potential bottleneck in your proposed architecture if GreenThumb AI scales to process data from 10,000 drones simultaneously, and how you would re-architect for that scale."
  • "How might an **AI Agent** assist you in drafting the initial proposal outline, and what specific technical details did you have to critically verify from its output?"

2. Team Project: Cloud AI Deployment Pipeline (40%)

Example Scenario: Automated AI Model Deployment

Prompt: Your team is tasked with building a fully automated Continuous Integration/Continuous Deployment (CI/CD) pipeline for an existing machine learning model (provided). The pipeline should:

  • Automatically retrain the model on new data (simulated hourly/daily updates) in the cloud.
  • Evaluate its performance (using provided metrics script).
  • Deploy a new version to a cloud endpoint only if performance improves, with rollback capability.
  • Trigger alerts for pipeline failures or performance degradation.

Your Project Portfolio must include the functional pipeline, comprehensive documentation (including **architecture diagrams**), and individual reflections on challenges and **AI Agent** usage.

Why AI-Resistant? Requires hands-on integration of multiple cloud services, CI/CD tools, and ML operations (MLOps) principles. AI can generate code snippets, but building, integrating, debugging, and documenting a complete, functional automated pipeline for a specific cloud platform is a complex engineering task.

Example Viva Questions (Post-Submission)

  • "Your pipeline used [Specific Cloud CI/CD Service]. What was the most challenging aspect of integrating the model retraining step with this service, and how did your team overcome it?"
  • "If the model performance dropped significantly after a new deployment, what debugging steps, using your **Hands-On Skills (HOS)**, would you take immediately to diagnose the issue?"

3. Programming Exercises (PE) (20%)

Example Scenario: Serverless AI Inference Endpoint

Prompt: Given a pre-trained small AI model (e.g., for image classification or text sentiment), deploy it as a serverless inference endpoint on a chosen cloud platform (e.g., AWS Lambda + API Gateway, Azure Functions). Your task is to:

  • Implement the serverless function to load and execute the model.
  • Configure the API endpoint for invocation.
  • Submit the code, deployment configuration files, and a **Prompt Engineering Log & Reflection** detailing your use of **AI Agents** to generate deployment scripts, YAML configurations, and Python code, along with your critical evaluation of their accuracy and suitability for the cloud environment.

Why AI-Resistant? Focuses on practical cloud deployment skills, critical evaluation of AI-generated configurations (which are often subtly incorrect or incomplete for specific cloud environments), and debugging the integration between different cloud services. The human must understand the nuances of the cloud provider's ecosystem.

Example PE-Related Discussion Questions (Class Forum / Instructor Check-in)

  • "Describe a challenge you faced when using an **AI Agent** to generate a specific cloud configuration (e.g., IAM roles, API Gateway proxy setup), and how your understanding of cloud **Hands-On Skills (HOS)** helped you correct it."
  • "What prompt did you use to generate the initial serverless function code, and how did you refine it after the first attempt failed?"
