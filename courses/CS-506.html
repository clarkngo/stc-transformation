CS 506: Programming for Computing - Proposed Assessments

Course Description: A foundational course focusing on modern programming concepts, data structures, and algorithms essential for advanced computing and specialized AI/Data Science work.

1. Applied Case Studies / Conceptual Assessment (40%)

Example Scenario: Algorithm Selection for a Social Media Feed

Prompt: A new social media app needs to design the core logic for its user feed. As the lead developer, you must evaluate two different data structures to manage incoming posts: a **priority queue** (based on "likes") and a **doubly linked list** (chronological). Prepare a technical trade-off analysis (4-5 pages) and a pseudo-code implementation for both approaches. Your analysis must:

  • Analyze the Big O time complexity for key operations (inserting a post, retrieving the top 10 posts) for each structure.
  • Discuss the memory usage implications of each choice.
  • Recommend a final structure based on the app's primary goal (engagement vs. recency) and justify your decision.

Why AI-Resistant? Requires critical analysis and justification of trade-offs between two valid solutions for a specific business problem. While an AI can define data structures, it struggles to perform a nuanced comparative analysis tied to a specific product goal and defend the choice.

Example Viva Questions (Post-Submission)

  • "You chose the priority queue to maximize engagement. In a real-world scenario, how would you handle the 'starvation' problem, where new posts never get seen? What algorithmic modification would you propose?"
  • "Walk me through the most complex part of your pseudo-code for the doubly linked list. What specific edge case did you have to account for?"

2. Team Project: Building a Small Command-Line Application (40%)

Example Scenario: A Simple Inventory Management System

Prompt: Your team will design and build a command-line application for a small bookstore to manage its inventory. The system must:

  • Utilize appropriate data structures (e.g., hash maps for book lookup, trees or lists for sorting).
  • Support core CRUD operations (Create, Read, Update, Delete) for books.
  • Persist data to a file (e.g., JSON or CSV).

Your Project Portfolio must include the functional application, a **UML class diagram** illustrating your object-oriented design, comprehensive source code, and individual reflections on debugging challenges and your use of **AI Agents** to write boilerplate code.

Why AI-Resistant? Requires translating business requirements into a coherent object-oriented design and implementing it. AI can generate snippets, but designing the overall class structure, managing state, and ensuring data persistence correctly demands human architectural thinking and debugging skills.

Example Viva Questions (Post-Submission)

  • "Explain your team's choice of a hash map versus a balanced binary tree for the primary book storage. What were the performance trade-offs?"
  • "Describe a bug that was difficult to find. What specific **Hands-On Skills (HOS)** in debugging did you use that your **AI Agent** couldn't help with?"

3. Programming Exercises (PE) (20%)

Example Scenario: Implementing a Custom Data Structure

Prompt: Implement a "Least Recently Used" (LRU) Cache from scratch using a combination of a hash map and a doubly linked list. Your task is to:

  • Build the data structure with `get` and `put` methods that run in O(1) time complexity.
  • Write a suite of unit tests to verify its correctness.
  • Submit your implementation, the tests, and a Prompt Engineering Log & Reflection detailing how you used an **AI Agent** to understand the core algorithm, generate initial class structures, and write test cases.

Why AI-Resistant? Focuses on implementing a non-trivial, combined data structure from first principles. The student must understand the underlying logic deeply enough to debug the interactions between the two components, a task where AI-generated code often fails subtly.
