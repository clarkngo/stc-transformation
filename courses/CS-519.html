CS 519: Cloud Computing - Proposed Assessments

Course Description: Provides a comprehensive understanding of cloud computing principles, service models (IaaS, PaaS, SaaS), deployment models, and key cloud technologies from major providers (AWS, Azure, GCP).

1. Applied Case Studies / Conceptual Assessment (40%)

Example Scenario: Disaster Recovery Plan for a Hybrid Cloud Environment

Prompt: A mid-sized financial company operates a critical legacy database on-premises but uses a public cloud for its customer-facing web applications. Design a comprehensive **Disaster Recovery (DR) plan and architecture diagram** for this hybrid cloud environment, ensuring RTO (Recovery Time Objective) of 4 hours and RPO (Recovery Point Objective) of 1 hour. Your proposal must:

  • Identify specific cloud services (e.g., replication, backup, failover) for both the on-premise and cloud components.
  • Outline the key steps and technologies for failover and failback procedures.
  • Analyze the cost implications of your proposed DR solution across different cloud providers.

Why AI-Resistant? Requires applying cloud principles to a specific, complex hybrid scenario with strict RTO/RPO constraints. While AI can list DR services, designing a coherent, cost-analyzed plan that integrates on-premise and cloud, and justifying specific technology choices, demands human expertise and **Hands-On Skills (HOS)** in cloud architecture.

Example Viva Questions (Post-Submission)

  • "You chose [Specific Cloud Service] for database replication. Explain the trade-offs between 'active-passive' and 'active-active' DR strategies for this component, and why your choice aligns with the RPO."
  • "What specific **Hands-On Skills (HOS)** (e.g., network configuration, firewall rules) would be most critical when testing the failover process for your web applications?"
  • "How might an **AI Agent** assist you in researching different DR strategies, and what part of its output (e.g., specific service configurations) did you need to critically verify for accuracy in a hybrid environment?"

2. Team Project: Multi-Cloud Web Application Deployment (40%)

Example Scenario: Resilient E-commerce Platform

Prompt: Your team is tasked with designing and partially implementing a highly available and resilient e-commerce platform that leverages a **multi-cloud strategy** (e.g., deploying front-end on AWS, backend on Azure, or specific services on different clouds). The platform should:

  • Host a simple web application (e.g., product catalog).
  • Demonstrate failover capabilities between cloud regions or providers.
  • Utilize containerization (e.g., Docker, Kubernetes) for deployment.

Your Project Portfolio must include the functional application, detailed **architecture diagrams** (showing cross-cloud communication), Infrastructure as Code (IaC) scripts, and individual reflections on challenges and **AI Agent** usage.

Why AI-Resistant? Requires hands-on integration of services across *multiple* cloud providers, which involves complex networking, security, and deployment challenges. While AI can generate IaC for a single cloud, designing, implementing, and debugging a multi-cloud solution that achieves resilience demands significant human expertise and **Hands-On Skills (HOS)** in cloud engineering.

Example Viva Questions (Post-Submission)

  • "Your multi-cloud design involves data synchronization between providers. What was the most significant challenge in maintaining data consistency, and what specific **Hands-On Skills (HOS)** (e.g., database administration, API integration) did your team use to address it?"
  • "Explain a scenario where a single-cloud deployment would have been insufficient for your e-commerce platform's resilience goals, and how your multi-cloud approach specifically mitigates that risk."

3. Programming Exercises (PE) (20%)

Example Scenario: Serverless API Gateway with Authentication

Prompt: Deploy a simple serverless API (e.g., a "Hello World" function) on a chosen cloud platform (e.g., AWS Lambda, Azure Functions) and secure it using a cloud-native authentication mechanism (e.g., API Gateway custom authorizer, Azure AD). Your task is to:

  • Implement the serverless function and API endpoint.
  • Configure the authentication mechanism.
  • Submit the code, configuration files, and a **Prompt Engineering Log & Reflection** detailing your use of **AI Agents** to generate code for the serverless function, authentication logic, and deployment scripts. Include your critical evaluation of the AI's suggestions for security and correctness.

Why AI-Resistant? Focuses on practical cloud security and serverless deployment, where subtle misconfigurations can lead to significant vulnerabilities. The student must apply **Hands-On Skills (HOS)** to critically evaluate AI-generated security configurations, debug authentication flows, and ensure the API is correctly secured according to best practices.

Example PE-Related Discussion Questions (Class Forum / Instructor Check-in)

  • "Describe a specific instance where an **AI Agent's** suggestion for implementing authentication contained a security flaw (e.g., exposed credentials). How did your **Hands-On Skills (HOS)** in cloud security best practices help you identify and correct it?"
  • "What prompt did you use to generate the initial authentication code, and how did you refine it after the first attempt to meet your security requirements?"
