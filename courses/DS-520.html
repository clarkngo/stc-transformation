DS 520: Data Mining - Proposed Assessments

Course Description: Explores core data mining techniques for pattern discovery, association rule learning, clustering, classification, and anomaly detection. Emphasizes real-world application and interpretation of results.

1. Applied Case Studies / Conceptual Assessment (40%)

Example Scenario: Fraud Detection in Credit Card Transactions

Prompt: A regional bank needs to enhance its fraud detection system for credit card transactions. Using a provided (simulated and anonymized) transaction dataset that includes known fraudulent activities, design a data mining pipeline. Prepare a technical report (7-9 pages) and a visualization dashboard (e.g., Tableau, Power BI mock-up) for the bank's risk management team. Your report must:

  • Justify your choice of data mining techniques (e.g., anomaly detection, classification algorithms) for this specific problem.
  • Detail your feature engineering strategy and handling of imbalanced data.
  • Analyze the trade-offs between precision and recall in this high-stakes context.
  • Translate your findings into actionable recommendations for improving fraud detection.

Why AI-Resistant? Requires applying complex data mining techniques to a high-stakes, real-world problem with nuanced performance considerations (e.g., false positives vs. false negatives in fraud). While AI can generate code for models, the critical judgment in feature engineering, metric selection, and translating results into *actionable, risk-aware* business recommendations is human-driven.

Example Viva Questions (Post-Submission)

  • "You chose [Specific Data Mining Technique]. Explain a specific limitation of this technique in detecting novel fraud patterns, and what **Hands-On Skills (HOS)** (e.g., feature selection, outlier detection) were crucial in preparing your data for it."
  • "In the context of fraud detection, which is more critical: high precision or high recall, and how would your model's performance impact the bank's customers and bottom line?"
  • "How might an **AI Agent** assist you in exploring different data mining algorithms for fraud detection, and what part of its output (e.g., assumptions, limitations) did you critically verify for relevance to financial data?"

2. Team Project: Building an Intelligent Recommendation System (40%)

Example Scenario: Hyper-Personalized Book Recommendations

Prompt: Your team is tasked with building a book recommendation system for a small, independent online bookstore. The system should:

  • Utilize historical user-book interaction data (provided, or publicly available).
  • Implement at least two distinct data mining approaches (e.g., collaborative filtering, content-based filtering, association rule mining).
  • Provide a demonstrable API endpoint that returns recommendations for a given user or book.

Your Project Portfolio must include the functional system, detailed **architecture diagrams** (showing data flow, model serving), comprehensive code, and individual reflections on challenges and **AI Agent** usage.

Why AI-Resistant? Requires implementing and integrating complex data mining algorithms, managing diverse data, and evaluating performance in a specific application (recommendation). While AI can generate code for individual algorithms, the architectural design, integration, and critical comparison of different data mining approaches for a specific business goal demand human expertise and **Hands-On Skills (HOS)**.

Example Viva Questions (Post-Submission)

  • "Your system uses [Specific Recommendation Algorithm]. What was the most challenging aspect of handling 'cold start' users/items (new users with no history), and how did your team address it using specific data mining concepts?"
  • "Explain the trade-offs your team considered between accuracy and diversity in your recommendations, and how that impacts the user experience for a small bookstore."

3. Programming Exercises (PE) (20%)

Example Scenario: Implementing and Optimizing a Clustering Algorithm

Prompt: Given a dataset of customer demographics, your task is to:

  • Implement a clustering algorithm (e.g., K-Means, DBSCAN) from a Python library to segment customers.
  • Experiment with different parameter settings (e.g., number of clusters, epsilon for DBSCAN) and evaluate the quality of the clusters using appropriate metrics (e.g., Silhouette score).
  • Interpret the characteristics of the discovered clusters to describe distinct customer segments.
  • Submit the code, a report on cluster characteristics, and a **Prompt Engineering Log & Reflection** detailing your use of **AI Agents** to explore clustering algorithms, generate code, and interpret cluster profiles. Include your critical evaluation of the AI's suggestions and any corrections you made.

Why AI-Resistant?** Focuses on the iterative process of applying and optimizing data mining algorithms, where human judgment is critical for parameter tuning and interpreting results. While AI can generate code, the student must apply **Hands-On Skills (HOS)** in data visualization and domain knowledge to discern meaningful clusters and justify their findings.

Example PE-Related Discussion Questions (Class Forum / Instructor Check-in)

  • "Describe a specific instance where an **AI Agent's** suggestion for the number of clusters led to poor results, and how your **Hands-On Skills (HOS)** (e.g., elbow method, domain knowledge) helped you choose a better parameter."
  • "What prompt did you use to ask the AI to 'characterize the differences' between your discovered customer segments, and how did you refine its output with your own domain understanding?"
