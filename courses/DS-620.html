DS 620: Machine Learning & Deep Learning - Proposed Assessments

Course Description: A rigorous exploration of fundamental machine learning algorithms and the theory and application of modern neural networks (Deep Learning).

1. Applied Case Studies / Conceptual Assessment (40%)

Example Scenario: Model Selection for Medical Imaging

Prompt: A hospital wants to develop a model to classify X-ray images into 'normal', 'pneumonia', and 'COVID-19'. As the lead ML engineer, you must evaluate two approaches: a **classic ML model** (e.g., SVM with engineered features) and a **Deep Learning model** (a Convolutional Neural Network - CNN). Prepare a model selection proposal (7-9 pages) with a risk analysis matrix. Your proposal must:

  • Analyze the pros and cons of each approach regarding performance, data requirements, computational cost, and interpretability.
  • Propose a rigorous validation strategy to compare the models.
  • Assess the ethical risks associated with misclassification for each category and recommend a final model.

Why AI-Resistant? Requires a deep, comparative analysis of two different modeling paradigms for a high-stakes application. The critical thinking lies in evaluating the trade-offs (e.g., is the higher accuracy of a "black box" CNN worth the loss of interpretability from an SVM?), which an AI cannot do without significant human guidance and domain context.

Example Viva Questions (Post-Submission)

  • "You highlighted 'interpretability' as a key concern. Using a specific **Hands-On Skill (HOS)** from an earlier course (like Explainable AI), how would you attempt to make the CNN model's decisions more transparent to a clinician?"
  • "Explain the concept of 'overfitting' in the context of your proposed CNN and what specific regularization techniques you would implement to mitigate it."

2. Team Project: Building and Deploying a Deep Learning Model (40%)

Example Scenario: Real-Time Object Detection

Prompt: Your team will build and deploy a real-time object detection model (e.g., using YOLO or a similar architecture) that can identify a small set of custom objects (e.g., different types of recyclable materials). The project involves:

  • Collecting and annotating a small, custom dataset.
  • Fine-tuning a pre-trained object detection model on your custom data.
  • Deploying the model as a simple web application that can process uploaded images or a live webcam feed.

Your Project Portfolio must include your annotated dataset, the trained model, the functional web app, and individual reflections detailing your team's approach to data augmentation, hyperparameter tuning, and use of **AI Agents** for debugging.

Why AI-Resistant? This is an end-to-end project that requires a wide range of practical skills: data collection/annotation, model fine-tuning, and deployment. While AI can help with code snippets, orchestrating the entire MLOps lifecycle—especially the messy, iterative process of data collection and model tuning—is a significant human-led engineering challenge.

Example Viva Questions (Post-Submission)

  • "Your team had to collect custom data. What was the biggest challenge in ensuring data quality and annotation consistency, and what specific **Hands-On Skill (HOS)** did you use to address it?"
  • "Walk me through the process of debugging your model when it failed to detect one of your custom objects. What did you change in your data or training process?"

3. Programming Exercises (PE) (20%)

Example Scenario: Implementing a Neural Network Layer from Scratch

Prompt: Implement a single 'fully connected' layer of a neural network from scratch in Python (using only NumPy), including both the forward and backward propagation steps. Your task is to:

  • Write the class for the layer, including methods for `forward(input)` and `backward(output_gradient)`.
  • Derive and implement the gradient calculations for weights and biases.
  • Write unit tests to verify the correctness of your gradients (e.g., using numerical approximation).
  • Submit your code, tests, and a Prompt Engineering Log & Reflection detailing how you used an **AI Agent** to help derive the backpropagation math and debug your NumPy code.

Why AI-Resistant? Focuses on the fundamental mathematical and programming building blocks of deep learning. While an AI can provide a complete solution, the student must understand the calculus and linear algebra well enough to debug the subtle errors that often arise in a from-scratch implementation, a critical **Hands-On Skill (HOS)**.
