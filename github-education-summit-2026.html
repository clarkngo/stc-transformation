Session 1 · GenAI in CS Education Consortium

The Academia–Industry Gap Is Not New — But It's More Urgent

Speaker: GenAI in CS Education Consortium (UCSD, funded by Google.org)

Core Tension

Faculty are divided on AI in the classroom

Views range from "students should only use AI for things they can already do themselves" to "if we don't change our curriculum we'll be as relevant as learning ancient Greek." The Consortium exists to support faculty who want to change but face real barriers — skills gaps, assessment design challenges, and lack of recent industry experience.

What's Changing

Less emphasis on writing code from scratch

Shared courses shift learning goals toward specification, verification, and debugging — the skills that matter when AI is doing the generation. Students use GenAI tools to complete take-home projects rather than writing everything by hand.

"Software creation with GenAI has changed more than I'd realized, and the skills to write software with these new tools are still a bit unclear."
— Consortium researcher, closing reflection
"We need close academia/industry collaborations more than ever."
— Consortium closing slide

Origin Story

25 years of writing code — then a full stop

Challen spent 25 years writing code by hand four days a week. Since July 2025 he no longer reads, writes, or debugs code by hand — and he's building more than ever. This personal shift drove him to redesign his course from the ground up.

The Course

No prerequisites, no code

An AI course for everyone. No code mental models required. Students engage with AI through conversation, not syntax.

The Collaboration

Claude as co-instructor

Every line of the course site, syllabus, rubrics, assessments, and annotated readings was written through conversational AI. Claude is content, infrastructure, and operations.

Assessment Design

Conversational, multi-agent, autograded

Students complete Turing Test conversations and other AI-driven assessments. A hidden second agent scores engagement in real time so the instructor sees readiness levels before class starts. Feedback is near-instant — the value of feedback decays rapidly after submission, so autograding everything matters.

Pedagogy

No lecturing — agents run the room

Before class: structured prep chats build context and surface readiness. During class: AI agents facilitate small-group discussion, surface patterns across the room, and signal when a group is ready to share. Humans do the thinking; the AI does the structural work.

Hypothesis

AI collaboration is its own distinct skill

Not "classical programming → AI on top." AI collaboration is independent from but related to classical programming — like studio art and art history. Either can come first; an artist learns both regardless. This framing changes how you sequence and assess learning.

What's Next

Fall 2026: Conversational Programming

"If you can talk it, you can create it." The next evolution of the course leans fully into the idea that natural language is the new programming interface — not a shortcut, but the actual medium.

Think of it like Roblox as a sandbox — a creative environment where you describe what you want to build and the platform handles the execution. The skill is imagination and specification, not syntax.

PRD as the New Code

The product requirements document is now a primary artifact

In a conversational programming model, students write PRDs (product requirements documents) to specify what they want built — articulating the problem, the user, the constraints, and the success criteria. The AI handles generation. This shifts assessment toward clarity of thought and problem framing, not implementation detail.

Grace Hopper saw this coming

Grace Hopper's vision was that normal human language — not machine code — should be the interface for building software. Her work on compilers was about making software creation accessible to people who think in words, not bits. Conversational programming is the next step in that same arc: if you can describe the problem clearly, you can build the solution.

"I dislike the term 'vibe coding' because it implies a certain degree of not caring about the results. I'm trying to establish conversational programming as an alternative."
— Geoffrey Challen, live Q&A
"It's not a purist issue. Geoff literally said 'don't look at the code.' We're offloading all of the cognitive work to an AI with this methodology."

Session 3 · GitHub Education × Major League Hacking (MLH)

Software Is Having Its iPhone Moment

The case for hackathons as the new classroom for AI skills.

The Big Claim

Coding is becoming a universal skill

The same democratization that happened to photography (85B photos in 2000 → 2.1 trillion today, a 25× increase) is happening to software. What was once a walled garden requiring specialized equipment and professional training is becoming commoditized, faster than humans, and basically free.

The Problem

97% of the future AI market doesn't exist yet

Daily AI users are in the 0.01% of early adopters. Most people's opinions on AI are based on experiences from 6+ months ago, used on contrived problems that don't actually apply. The gap between early adopters and the rest of the market represents enormous educational opportunity.

The Shift

When the cost of code goes to zero, ideas, taste, and distribution win

Aka: judgment, connection, and trust. Hackathons aren't just for developers anymore — writing prompts that produce results, interviewing users to build context, and sketching interfaces people want to use are now valid hackathon contributions.

Prompts vs. Agents

Know the difference

A prompt is a one-shot instruction that lives and dies in the chat. An agent is a saved workflow with a job description — context preloaded, multi-step, reusable by your whole team.

Better Prompts

Chain small steps, not one big ask

One big prompt → inconsistent, mediocre output. Five small prompts (Research → Outline → Draft → Critique → Polish) → consistent, better output. Decompose before you generate.

Cross-Cutting Themes

The Hard Questions the Room Was Wrestling With

From the live Q&A and attendee chat — the tensions that didn't resolve neatly.

Unresolved

Will AI eliminate junior developer roles?

If AI can automate entry-level tasks, does integrating GenAI into academia help students enter a shrinking market, or does it equip them to operate at a higher level from day one? The room had no consensus — but most agreed the question demands an answer from educators.

Unresolved

Are students actually learning — or just shipping?

Conversational programming gets things built faster. But one attendee challenged: "We're offloading all cognitive work. Does it let students make things more quickly with less effort? Yes. Do they actually learn as much? No way." The pedagogy is genuinely open.

Community Voice

Educators are feeling lost — and that's honest

One attendee summed it up plainly: "I'm so discouraged as a CS educator. We don't know the skills we need to teach, and we don't even know what those skills are." The response wasn't to dismiss the concern — it was to build advisory relationships with industry employers to stay calibrated.

Structural Problem

CS educators struggle to teach Level 1

The hardest layer to teach is Layer 1 — the problem-focused builder who uses AI to solve domain problems without deep engineering knowledge. Traditional CS curricula are built around Layers 3 and 4. Reaching Level 1 thinkers — the nurses, the lawyers, the small business owners — requires a completely different pedagogical approach that most CS educators weren't trained for.

We are problem solvers — not programmers

The most resonant reframe from the day: identifying as a programmer constrains how much you think you can impact the world. Identifying as a problem solver opens every domain. The question isn't "can I code this?" — it's "what problem am I solving, and for whom?" AI handles the translation from intent to implementation. This identity shift is as much a curriculum design challenge as a technical one.

"CS is becoming ancillary to any field — bioengineering, chemistry, biology, law — where critical thinking will be important. The emphasis on fundamentals like networking, cybersecurity, system design, and architecture is more important, not less."

So What

Implications for STC

How these ideas connect to the work we're already doing.

🎯

Reframe learning goals around verification, not generation

Less emphasis on writing from scratch. More emphasis on specifying what you want, critiquing what AI produces, and debugging when it's wrong.

📐

Assessment redesign is the hardest part

Conversational, multi-agent, autograded feedback is where Challen's approach is most ahead. Worth piloting even one assignment in this mode.

🛤️

Use the Curious → Native spectrum as a placement tool

MLH's 5-level path (Curious / User / Builder / Operator / Native) maps cleanly to how we could differentiate instruction or advise students on next steps.

🤝

Industry advisory ties aren't optional anymore

Multiple voices at the Summit pushed this. If we don't know what skills employers need in 12–24 months, we're designing curriculum for the past.

🔓

Hackathons as accessible AI labs

Non-developers belong in hackathons now — prompt engineers, interviewers, UX sketchers. Could be a low-barrier way to introduce AI to non-CS students.

🪪

The Microsoft Agentic AI cert is worth tracking

A formal, stackable credential for building agentic AI systems. Could anchor a certificate program or serve as an industry validation alongside our own courses.
