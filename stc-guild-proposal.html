The STC Guild

A project-based framework designed to solve the Turnover Trap and build lasting institutional memory.

01. The Turnover Trap

In technical programs, students often graduate in 2–4 years. When a club is built around a single charismatic leader or a niche topic, it collapses when they leave.

"The learning curve is steep, and by the time a leader masters the role, they graduate."

Symptoms of Decay

  • ✕ Fragmented leadership
  • ✕ "Ghost" clubs with no activity
  • ✕ Lost documentation and code

The Solution: Hub-and-Spoke

The STC Hub (Core)

One robust administrative structure. Handles the "boring" but vital tasks: room bookings, funding, recruitment, and platform management.

The Project Spokes

Modular, rotating projects. Instead of maintaining a "Club Identity," students focus on hands-on work with clear start and end dates.

02. Ensuring Survival

GitHub as History

All projects live in a central organization. A student in 2026 can resume a 2024 project without starting from zero.

HOS Alignment

Align projects with "Hands-on Skills" sessions. This creates a natural pipeline from learning to doing.

Low-Stakes Roles

Recruiting a "Project Lead" for a 3-month sprint is easier than finding a year-long "President."

The STC Guild Strategy

The Core

One board to handle admin and "The Byte" newsletters.

Project Tracks

Internal Link Dashboards, AI Coding Assistants, and Cyber Sprints.

Portfolio Focus

READMEs and Handover Docs are mandatory for every project.

Pro Tip

Focus on "Legacy Projects." Building tools the school actually uses (like a lab tracker) creates a built-in incentive for future cohorts to maintain them.
